## 三、开发问题实录

### 3.1 并发写入锁升级

**问题：** 20线程并发写入测试通过，但3读+2写线程并发时报 `TypeError: type NoneType doesn't define __round__ method`

**排查：** `get_statistics()` 中 `row["total_box_office"]` 返回了 None，虽然 SQL 有 COALESCE。原因是同一 `sqlite3.Connection` 被多线程并发访问存在底层竞争。

**解决：** 将 `_write_lock` 升级为 `_db_lock`，所有读写操作统一串行化。

### 3.2 数据库版本误升级

**问题：** `check_and_migrate()` 把版本从 "1.0" 升级到 "1.1"

**排查：** 迁移字典包含空 SQL 列表的 "1.1" 条目，循环无条件执行了升级。

**解决：** 增加 `if not sqls: continue` 跳过无实际迁移项的版本。

### 3.3 pyecharts API 兼容

**问题：** 图表生成时报 `AxisPointerOpts.__init__() got an unexpected keyword argument 'type'`、`GridOpts.__init__() got an unexpected keyword argument 'left'`

**原因：** pyecharts 2.1.0 API 与旧版不一致，`GridOpts` 参数为 `pos_left`/`pos_right`/`pos_top`/`pos_bottom`。

**解决：** 移除不兼容参数，使用兼容的子集。

### 3.4 QWebEngineView 初始化顺序

**问题：** `ImportError: QtWebEngineWidgets must be imported before a QCoreApplication instance is created`

**解决：** 在 `main.py` 中 `QApplication` 创建之前就 `import QWebEngineView`。

### 3.5 中文字符编码

**问题：** 控制台输出 `[INFO] [Main] ϵͳ���� ...` 中文乱码

**原因：** Windows 终端 GBK 编码 vs Python 日志 StreamHandler 的 UTF-8 输出。

**解决：** `sys.stdout.reconfigure(encoding="utf-8")` 强制 UTF-8 输出。
