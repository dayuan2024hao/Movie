# 技术设计文档 v1.0

> 对应需求：电影票分析系统
> 技术栈：PyQt5 + PyECharts + Pandas + SQLite

---

## 1. 系统架构

### 1.1 三层架构

```
┌─────────────────────────────────────────────────┐
│                 表示层 (UI)                      │
│  PyQt5 主窗口 + 页面切换 + PyECharts 图表嵌入   │
├─────────────────────────────────────────────────┤
│                 业务层 (Logic)                   │
│  爬虫引擎 | 推荐引擎 | 筛选引擎 | 数据管理      │
├─────────────────────────────────────────────────┤
│                 数据层 (Data)                    │
│  SQLite 数据库 | CSV 备份 | 数据访问对象 (DAO)  │
└─────────────────────────────────────────────────┘
```

### 1.2 数据流

```
豆瓣/猫眼 ──→ 爬虫引擎 ──→ 数据清洗 ──→ SQLite
                              │
                          CSV 备份 (容灾)
                              │
                     ┌───────┴───────┐
                     │               │
                 数据分析         UI 展示
                     │               │
                  推荐算法 ←── 用户交互 (筛选/搜索)
                     │
                  推荐结果 → UI 展示
```

## 2. 技术选型详解

### 2.1 PyQt5

| 模块 | 用途 |
|------|------|
| `QMainWindow` | 主窗口框架 |
| `QStackedWidget` | 页面切换（看板/推荐/详情）|
| `QTableWidget` / `QTableView` | 电影数据表格 |
| `QListWidget` / `QTreeWidget` | 导航栏/筛选条件 |
| `QSlider` | 评分/票价区间滑块 |
| `QWebEngineView` | 嵌入 PyECharts 图表（HTML）|
| `QThread` / `QThreadPool` | 后台爬虫线程 |
| `QSS` | 界面样式美化 |
| `Qt Designer` | UI 布局拖拽设计（辅助开发）|

### 2.2 PyECharts

```
pyecharts.charts:
  - Bar          → 票房 Top 10 柱状图
  - Line         → 票房趋势折线图
  - Pie          → 类型占比饼图
  - Scatter      → 评分 vs 票房散点图
  - HeatMap      → 数据热度图（可选）

pyecharts.render:
  - render_embed() → 生成 HTML 嵌入 QWebEngineView
```

### 2.3 数据层

```
sqlite3 (Python 内置)
  - 无需安装
  - 单文件存储: data/movie_analysis.db
  - 线程安全：每个连接独立 cursor
  - 支持事务
```

## 4. 关键技术决策

### 4.1 为什么用 QWebEngineView 嵌入 PyECharts？
- PyECharts 图表的交互性远强于 Matplotlib
- 图表支持：悬停提示、缩放、数据视图、导出图片
- 但需要嵌入浏览器组件 (QWebEngineView ≈ 30MB)

### 4.2 为什么用 SQLite？
- 桌面应用无需数据库服务器
- 单文件便于打包和分发
- Python 内置 sqlite3 模块
- 项目数据量 < 1 万条，SQLite 完全胜任

### 4.3 爬虫为什么做容灾？
- 网络请求不可靠，爬虫可能失败
- CSV 备份保证核心功能演示不受影响
- 爬虫状态对用户可见：成功/失败/进行中

### 4.4 线程模型
- 主线程：UI 渲染和用户交互
- 后台线程：爬虫数据采集
- 使用 QThread + 信号(signal) 机制更新 UI（线程安全）

## 5. 数据流时序

### 5.1 启动流程
```
应用启动
  ↓
加载数据库 → 检查数据是否存在
  ↓                    ↓
数据存在            数据不存在
  ↓                    ↓
加载到 UI        触发爬虫/加载备份
  ↓                    ↓
显示看板         显示数据加载中
```

### 5.2 筛选流程
```
用户调整筛选条件
  ↓
触发筛选事件 (300ms 防抖)
  ↓
查询数据库 (带 WHERE 条件)
  ↓
更新统计卡片
  ↓
更新所有图表
  ↓
更新数据表格
```

### 5.3 推荐流程
```
用户进入推荐页
  ↓
选择推荐模式 (综合/热门/高分/高性价比)
  ↓
获取当前筛选范围内的电影数据
  ↓
计算综合评分
  ↓
按评分降序排列
  ↓
生成推荐理由
  ↓
渲染推荐卡片列表
```

## 6. 错误处理策略

| 场景 | 处理方式 |
|------|---------|
| 数据库文件损坏 | 自动重建 + 从 CSV 恢复 |
| 爬虫网络错误 | 重试 3 次 → 失败提示 + 使用缓存 |
| 图表渲染失败 | 显示空白占位 + 错误日志 |
| UI 操作异常 | 弹窗提示 + 不崩溃 |
| 数据为空 | 显示友好的空状态引导 |

## 7. 打包部署

```bash
# 安装 PyInstaller
pip install pyinstaller

# 打包命令
pyinstaller --name="电影票分析系统" ^
            --windowed ^
            --onefile ^
            --add-data "data;data" ^
            --add-data "resources;resources" ^
            --icon="resources/icon.ico" ^
            main.py

# 输出: dist/电影票分析系统.exe
```

> 注意：--onefile 会导致启动变慢，但提交作业更方便。
> 如果文件太大（>500MB），改用 --onedir 模式。
