# 详细开发步骤 v1.0

> 本文档将整个开发过程分为 8 个阶段，每个阶段有明确的目标、完成标准和预计耗时。
> ⚠️ **每个阶段完成后，必须经过验证才能进入下一阶段。**

---

## 阶段总览

| 阶段 | 名称 | 预计耗时 | 前置依赖 |
|------|------|---------|---------|
| **0** | 环境搭建与项目初始化 | 1 天 | 无 |
| **1** | 数据层：数据库 + 数据准备 | 2 天 | 阶段 0 |
| **2** | UI 框架：主窗口 + 导航 + 页面容器 | 2 天 | 阶段 1 |
| **3** | 可视化看板：图表 + 统计卡片 + 表格 | 3 天 | 阶段 2 |
| **4** | 推荐系统：算法 + 推荐榜页面 | 2 天 | 阶段 3 |
| **5** | 搜索与筛选功能 | 2 天 | 阶段 4 |
| **6** | 电影详情页面 | 1 天 | 阶段 5 |
| **7** | 综合测试 + 打包 + 文档 | 2 天 | 阶段 6 |

> **总预计耗时：约 15 天**（可根据实际进度调整）

## 阶段 0：环境搭建与项目初始化

### 目标
搭建开发环境，创建项目骨架，确保基础运行正常。

### 任务清单

- [ ] **0.1 安装 Python 3.10+**
  - 确认 `python --version` ≥ 3.10
  - 确认 `pip --version` 正常

- [ ] **0.2 安装项目依赖**
  ```bash
  pip install PyQt5 PyQtWebEngine pyecharts pandas numpy requests beautifulsoup4 lxml
  ```
  - 测试 `import PyQt5` 无报错
  - 测试 `from pyecharts.charts import Bar` 无报错

- [ ] **0.3 创建项目骨架**
  ```
  movie_analysis_system/
  ├── main.py                 # (占位，打印 Hello World)
  ├── database/__init__.py
  ├── crawler/__init__.py
  ├── ui/__init__.py
  ├── ui/widgets/__init__.py
  ├── charts/__init__.py
  ├── recommendation/__init__.py
  ├── data/                   # 空文件夹（放数据库和备份）
  ├── resources/
  │   ├── styles/
  │   └── icons/
  ├── tests/                  # 可选
  └── requirements.txt        # 生成: pip freeze > requirements.txt
  ```

- [ ] **0.4 验证环境**
  - 运行 `main.py` 能正常打印 "系统启动"
  - Node: 此阶段不涉及任何业务逻辑

### 完成标准
✅ `python main.py` 正常运行
✅ 所有依赖安装成功
✅ 项目目录结构完整
✅ requirements.txt 已生成

## 阶段 1：数据层 — 数据库 + 数据准备

### 目标
实现数据库管理模块，准备示例数据和爬虫脚本。

### 任务清单

- [ ] **1.1 实现 db_manager.py**
  - 单例模式管理数据库连接
  - `init_db()` 创建所有表
  - `insert_movie()` 插入/更新
  - `query_movies()` 带筛选条件查询
  - `get_statistics()` 统计卡片数据
  - `get_top10_box_office()` Top10 数据
  - `get_genre_stats()` 类型统计数据

- [ ] **1.2 准备备份数据**
  - 创建 `data/backup_movies.csv`
  - 包含 50-200 条真实电影数据（手动整理）
  - 字段与 movies 表一致
  - 编码：UTF-8

- [ ] **1.3 实现豆瓣爬虫（基础功能）**
  - `DoubanSpider.get_top250()` 爬取 Top250 列表
  - `DoubanSpider.get_movie_detail()` 爬取详情
  - 反爬策略：延迟 + User-Agent 轮换
  - 进度回调（用于 UI 进度条）

- [ ] **1.4 实现数据加载逻辑**
  - 启动时检查数据状态
  - 无数据 → 尝试爬虫 → 失败 → 加载 CSV 备份
  - 数据状态对 UI 可查

### 完成标准
✅ `db_manager.py` 所有查询方法正常
✅ 备份数据 CSV 可用
✅ 爬虫能成功爬取豆瓣 Top250
✅ 数据加载逻辑完整（自动降级）

## 阶段 4：推荐系统 — 算法 + 推荐榜页面

### 目标
实现综合评分推荐算法，设计醒目的推荐榜页面。

### 任务清单

- [ ] **4.1 评分模型 (recommendation/scorer.py)**
  - `ComprehensiveScorer` 类
  - 评分维度：rating_score (35%)
  - 票房维度：box_office_score (25%)
  - 热度维度：popularity_score (20%)
  - 性价比维度：value_score (20%)
  - 数据归一化处理
  - 缺失值权重分配

- [ ] **4.2 推荐逻辑 (recommendation/recommender.py)**
  - `Recommender` 类
  - `comprehensive()` — 综合推荐 (默认)
  - `hot()` — 热门推荐
  - `high_rating()` — 高分推荐
  - `best_value()` — 高性价比推荐
  - 推荐理由生成

- [ ] **4.3 推荐榜 UI (recommendation_page.py)**
  - 模式切换 Tab：综合/热门/高分/高性价比
  - 推荐卡片列表（大尺寸）
  - 前三名 🥇🥈🥉 特殊样式
  - 每张卡片：排名 + 电影信息 + 推荐理由
  - "查看详情"按钮
  - "显示更多"分页

- [ ] **4.4 推荐榜入口**
  - 统计卡片右侧增加"推荐榜"入口卡片
  - 点击跳转到推荐页

### 完成标准
✅ 推荐算法计算正确
✅ 4 种推荐模式各有不同结果
✅ 推荐理由合理
✅ 推荐榜 UI 醒目、大尺寸展示
✅ 前三名特殊样式

## 开发节奏建议

2. 选 2-3 个小任务完成
3. 每完成一个任务就运行测试

遇到困难：
1. 先自己尝试 15 分钟
