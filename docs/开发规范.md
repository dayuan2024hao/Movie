## 1. Python 编码规范

### 1.1 命名约定

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 / 模块名 | `snake_case` | `db_manager.py`, `douban_spider.py` |
| 类名 | `PascalCase` | `DatabaseManager`, `ComprehensiveScorer` |
| 函数 / 方法 | `snake_case` | `get_movie_detail()`, `calc_score()` |
| 变量 | `snake_case` | `movie_list`, `total_box_office` |
| 常量 | `UPPER_SNAKE` | `MAX_RETRIES = 3`, `CHART_THEME` |
| 私有成员 | `_snake_case` | `_parse_html()`, `_cache` |
| 保护成员 | `_snake_case` | `_internal_data` （少用）|

## 4. 错误处理规范

### 4.1 基本原则

```python
# ✅ 正确：捕获特定异常
try:
    data = spider.get_data()
except requests.RequestException as e:
    logger.error(f"网络请求失败: {e}")
    data = load_backup()
except ValueError as e:
    logger.error(f"数据解析失败: {e}")
    data = []
else:
    logger.info(f"成功获取 {len(data)} 条数据")
finally:
    spider.close()

# ❌ 错误：捕获所有异常（隐藏 bug）
try:
    data = spider.get_data()
except:
    data = []
```

### 4.2 自定义异常

```python
class CrawlerError(Exception):
    """爬虫模块异常基类。"""
    pass

class NetworkError(CrawlerError):
    """网络请求异常。"""
    pass

class ParseError(CrawlerError):
    """数据解析异常。"""
    pass

class DatabaseError(Exception):
    """数据库操作异常。"""
    pass
```

### 4.3 异常向上传递规则

```
爬虫模块 → 抛出 CrawlerError
          ↓
UI 层   → 捕获后弹窗提示 + 记录日志
          ↓
         → 不崩溃，显示友好错误界面
```

## 5. 日志规范

### 5.1 日志级别

| 级别 | 使用场景 |
|------|---------|
| DEBUG | 调试信息（开发环境）|
| INFO | 正常流程信息：爬虫开始/完成，页面切换 |
| WARNING | 异常但不影响运行：爬虫重试，使用缓存数据 |
| ERROR | 功能异常：爬虫彻底失败，查询失败 |
| CRITICAL | 程序无法继续运行（极少使用）|

### 5.2 日志格式

```
[2024-01-15 14:30:22] [INFO] [Crawler] 开始爬取豆瓣 Top250...
[2024-01-15 14:30:25] [WARNING] [Crawler] 第 3 次请求超时，正在重试 (1/3)
[2024-01-15 14:31:00] [INFO] [Crawler] 爬取完成，共获取 250 条记录
[2024-01-15 14:31:01] [ERROR] [Database] 数据写入失败: UNIQUE constraint failed
```

## 6. Git 规范（如使用）

### 6.1 分支管理

```
main        → 稳定版本（打包提交用）
dev         → 开发主分支
feature/*   → 功能分支 (feature/crawler, feature/recommend)
fix/*       → 修复分支 (fix/table-sort-bug)
```

### 6.2 Commit 规范

```
<type>(<scope>): <subject>

类型:
  feat    → 新功能
  fix     → 修复 bug
  docs    → 文档变更
  style   → 样式/格式变更（不影响代码逻辑）
  refactor→ 重构
  perf    → 性能优化
  chore   → 构建/工具变更

示例:
  feat(crawler): 实现豆瓣 Top250 爬虫
  fix(table): 修复表格排序不生效的问题
  docs: 更新需求分析文档 v1.0
  style(ui): 调整统计卡片阴影效果
```

## 8. 日常开发流程

### 8.1 每日开发步骤

```
1. 打开项目 → 查看 dev_log 中今日待办
2. 开始开发 → 遵循本规范
3. 完成功能 → 自测
4. 更新 dev_log → 记录完成事项和待办
5. 如使用 Git → commit
```

### 8.2 提交前检查清单

- [ ] 代码运行无报错
- [ ] 新功能已自测
- [ ] 遵守了编码规范（命名/注释/格式）
- [ ] 无调试用的 print/console.log
- [ ] 异常情况已处理（try/except）
- [ ] 更新了开发日志
- [ ] 数据库变更考虑向后兼容

## 9. 协同工作规范（AI 辅助）

当使用 Claude 等 AI 辅助开发时：
