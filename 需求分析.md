# 🎬 电影票分析系统 — 需求分析文档 v1.0

## 一、项目概述

### 1.1 项目目标
开发一个运行在 Windows 平台上的电影票分析系统，通过网络爬虫获取电影数据，综合**电影票房、电影评价、电影评分、电影票价格**等因素进行数据分析与可视化，并给出智能电影推荐。

### 1.2 用户定位
- 电影爱好者 / 观影决策者
- 系统帮助用户发现值得观看的电影

### 1.3 项目性质
- 学校实训项目，用于课程提交评分

---

## 二、技术选型

| 层面 | 选型 | 说明 |
|------|------|------|
| **GUI 框架** | **PyQt5** | 桌面级应用，支持 Qt Designer 拖拽设计，界面专业 |

## 三、数据采集方案（得分点设计）

### 3.1 爬虫架构

```
豆瓣 Top250 ──→ 电影基本信息、评分、评价数、短评
     ↓
猫眼票房    ──→ 电影票房数据
     ↓
数据清洗    ──→ 合并、去重、缺失值处理
     ↓
SQLite 数据库 ←── CSV 备份文件（爬虫失败时自动加载）
```

### 3.2 豆瓣爬虫（主要得分点）

| 爬取内容 | 难度 | 说明 |
|---------|------|------|
| 电影名称、类型、导演、主演 | ⭐ 简单 | 公开页面直接解析 |
| 评分、评分人数 | ⭐ 简单 | 直接获取 |
| 上映日期、片长、地区 | ⭐ 简单 | 详情页获取 |
| 短评内容（可选） | ⭐⭐ 中等 | 用于情感分析加分项 |
| 海报图片 URL | ⭐ 简单 | 用于 UI 展示 |

**反爬策略**：
- 请求间隔 1-3 秒随机延迟
- User-Agent 轮换池
- 单次爬取最多 250 条（Top250），不会触发封 IP

### 3.3 猫眼票房爬虫（辅助得分点）

| 爬取内容 | 难度 | 说明 |
|---------|------|------|
| 实时票房榜 | ⭐⭐ 中等 | 调用公开 API |
| 电影累计票房 | ⭐⭐ 中等 | 页面解析 |

### 3.4 票价数据方案
票价数据无公开接口，采用以下替代方案：
- **方案 A**：根据"票房 ÷ 预估观影人次"估算平均票价
- **方案 B**：少量手动录入典型票价作为样本
- **方案 C**：在推荐模型中将"性价比"维度替换为"热度指数"(票房 × 评价数)

### 3.5 容灾设计
- 爬虫模块与核心功能解耦
- **预置 CSV 示例数据**，爬虫失败自动加载备份数据
- 所有显示爬虫状态的 UI 提示
- **保证演示不翻车 ✅**

## 四、数据存储

### 4.1 数据库设计（SQLite）

**表 1：movies（电影基本信息）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| title | TEXT | 电影名称 |
| genre | TEXT | 类型（逗号分隔，如"动作,科幻"）|
| director | TEXT | 导演 |
| actors | TEXT | 主演（前 5 位） |
| release_date | TEXT | 上映日期 (YYYY-MM-DD) |
| runtime | INTEGER | 片长（分钟） |
| region | TEXT | 地区 |
| rating | REAL | 豆瓣评分 (0.0-10.0) |
| rating_count | INTEGER | 评分人数 |
| review_count | INTEGER | 短评数 |
| box_office | REAL | 票房（万元） |
| ticket_price | REAL | 平均票价（元） |
| poster_url | TEXT | 海报链接 |
| summary | TEXT | 剧情简介 |

**表 2：reviews（短评 - 可选加分项）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| movie_id | INTEGER | 关联电影 ID |
| content | TEXT | 短评内容 |
| sentiment_score | REAL | 情感分析得分 (-1.0 ~ 1.0) |

### 4.2 数据管理功能
- 启动时自动检查数据状态
- 手动"刷新数据"按钮重新爬取
- 数据统计概览（总电影数、数据时间范围）
- 导出数据为 CSV

## 七、电影推荐（核心功能，醒目设计）

### 7.1 推荐算法 — 综合评分模型

```
推荐分 = 评分 × 0.35 + 票房分 × 0.25 + 热度分 × 0.20 + 性价比分 × 0.20
```

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| **评分分** | 35% | rating / 10（归一化到 0-1）|
| **票房分** | 25% | box_office / max(box_office)（归一化）|
| **热度分** | 20% | review_count / max(review_count)（评价数归一化）|
| **性价比分** | 20% | rating / ticket_price（评分票价比，再归一化）|

> 注：对于票价缺失的数据，性价比分权重自动分配到评分和热度上

### 7.2 推荐模式 Tab

| Tab | 说明 |
|-----|------|
| ⭐ **综合推荐** | 按综合评分排序，默认展示 |
| 🔥 **热门推荐** | 按票房+热度排序 |
| 💎 **高分推荐** | 只看评分 9.0+ 的电影 |
| 🎯 **高性价比** | 评分票价比最高的电影 |
