# 电影票分析系统 — 项目工作指引

## 📁 项目概述

Windows 桌面端电影票分析系统，综合票房、评分、评价、票价等因素进行可视化分析和电影推荐。

- **技术栈**：PyQt5 + PyECharts + Pandas + SQLite
- **Python 版本**：3.10+
- **项目类型**：学校实训项目

| 3 | 可视化看板：图表 + 卡片 + 表格 | ✅ 已完成 |

## 📝 开发日志管理

- 日志文件：`dev_log/YYYY-MM-DD.md`
- 生成工具：`python dev_log/new_log.py`（自动创建今日日志）
- **每次开发结束时必须更新日志**，记录：
  - 今日完成事项
  - 遇到的问题 & 解决方案
  - 代码变更摘要
  - 明日待办

---

## ✅ 开发工作流

### 每次启动开发时：
1. 查看 `dev_log/` 中最近的日志，确认待办事项
2. 查看当前阶段的完成进度
3. 明确本次要完成的任务

### 每次完成开发时：
1. 更新 `dev_log/今日日期.md`
2. 确保代码符合 `docs/开发规范.md`
3. 代码能正常运行无报错

### 阶段切换时：
1. 对照 `docs/详细开发步骤.md` 的完成标准逐条检查
2. 确认当前阶段全部完成后再进入下一阶段
3. 更新阶段状态（在 CLAUDE.md 或日志中标记）

---

## 🔍 遇到问题时

1. 先搜索错误信息，尝试自己解决（15 分钟原则）
2. 在 `dev_log` 中记录问题
3. 可以向我（Claude）描述问题，我会协助分析
4. 如需修改设计文档，更新后保持文档与代码一致

---

## 📌 重要提醒

- **不要跳跃开发**：必须按照阶段顺序，前一阶段未完成不进入下一阶段
- **不要跳过验证**：每个阶段完成后必须验证通过
- **保持文档同步**：代码变更 → 文档更新
- **备份数据优先**：爬虫功能之前，必须先准备好 CSV 备份数据
- **SQLite 优先**：不要使用 MySQL，本项目为桌面单机应用
